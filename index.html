RESUMEN

HTML, es el lenguaje estándar que se utiliza para estructurar y presentar contenido en las páginas web. La estructura HTML es como el esqueleto de una página web. Comienza con una declaración del tipo de documento (DOCTYPE) que indica al navegador la versión de HTML que se está utilizando. A continuación, se encuentran las etiquetas < html>, que engloban todo el contenido de la página. Dentro de estas, se distinguen dos secciones principales: < head>, que contiene información meta como el título, codificación de caracteres y enlaces a hojas de estilo, y < body>, donde se coloca todo el contenido visible de la página, como encabezados, párrafos, imágenes, enlaces y otros elementos. Esta estructura jerárquica, definida por etiquetas de apertura y cierre, permite a los navegadores interpretar y mostrar el contenido de una manera organizada y coherente..

PROCESO DE DISEÑO



Ideas Generales


  • El proceso de diseño tiene varias fases para llevar acabo un proyecto web, planificacion, benchmarking, modelado de usuario y diseño conceptual.

  • La planificacion es la parte en donde se desarrolla el para que de la pagina, sus funciones, es mas que un analisis completo para el desarrolo de un proyecto

  • Benchmarking es hacer un sondeo de competidores con relacion de nuestra pagina web, en donde vemos como ellos han realizado su pagina web, en donde hacemos un analisis, para saber que aspecto mejorar y que aspectos podemos implentar

  • Modelado de usuario es la seccion en la que hacemos un analisis de las personas que son de nuestro mercado, indentificando las necesidades de los usuarios a los que va dirigidos para mejorar el enfoque de la pagina.

  • Diseño conceptual es una sketch de como queremos estrucutrar nuestra pagina web, para tener una orientacion de como queremos que esta se vista y para completar, es fundamental porque ya de ahi tendremos que hacer nuestro diseño final en cual este contrendra las imagenes y estilos y colores.

Apuntes de Clase


El proceso de diseño es muy importante a la hora de realizar un proyecto web, ya que este contendra toda la informacion recopilada para que el sitio web sea competitivo dentro del mercado, este constanta de varias fases para que el margen de error sea el minimo para la eficiencia a la hora de ejecutar el proyecto, estas fases recopilan informacion del usuarios, informacion de otros competidores y nos da una vision de como competir contra otras empresas desarroladoras web.



RESUMEN

El proceso de diseño implica una serie de etapas interconectadas que comienza con el benchmarking, donde se estudian los mejores ejemplos de la industria para identificar oportunidades de mejora. A continuación, se realiza un modelado detallado de los usuarios para comprender sus necesidades y comportamientos. Con esta información, se elabora un plan de diseño que define los objetivos, estrategias y tareas necesarias para crear una solución que satisfaga las necesidades de los usuarios y se destaque en el mercado. Este proceso iterativo garantiza que el diseño final sea relevante, útil y exitoso.

DISPLAY



Ideas Generales



  • Es una propiedad de CSS 2.1, que establece el tipo de caja de cada elemento

  • Cada elemento de HTML tiene un valor de visualización, por predeterminado viene Block o Inline

  • El elemento display:block;, siempre comienza en una nueva linea y ocupa todo el ancho disponible. Ejmplos donde esta propiedad: < div>, < h1 a h6>, < p>, < form>, < header>, etc..

  • Propiedad display:inline;, un elemento Inline no comienza en una nueva linea y solo ocupa el ancho necesario. Por ejemplo: < span>, < a> y < img>.

  • Propiedad display:inline-block;, permite establecer el ancho y una altura en el elemento, ademas con display inline-block; se respeta los padding y margin que display:inline; no son respetados

  • Esta propiedad no da un salto de linea despues del elemento, por lo que el elemento puede alinearse junto a otros elementos.

Apuntes de Clase



La propiedad es fundamental a la hora de estructurar un sitio web, porque la propiedad ayuda a acomodar cajas invisbles dentro del HTML para poder crear secciones ella mas facil, ya que esta asigna un tamaño especifico dependediendo la cantidad de objetos que quieres tener dentro de esta caja y con propiedadque utilices puede que ajuste tus objetos en una sola linea o haga un salto de linea, cabe aclarar que hay atributos que poseen una display predeterminado, como son los h1, p, etc ya que estas hacen un salto de linea por su nivel de jerarquia y otros propiedad puedes convivir dentro de una linea de texto.




RESUMEN

La propiedad CSS display es fundamental para determinar cómo se posicionan y comportan los elementos en una página web. Esta propiedad, introducida en CSS 2.1, define el tipo de caja que cada elemento ocupa. Por defecto, los elementos HTML tienen un valor de visualización block (comienzan en una nueva línea y ocupan todo el ancho disponible) o inline (no comienzan en una nueva línea y solo ocupan el ancho necesario). El valor inline-block combina lo mejor de ambos mundos, permitiendo establecer un ancho y altura definidos, además de respetar los márgenes y rellenos, a diferencia de los elementos inline. Esta propiedad es invaluable para crear diseños flexibles y personalizados, ya que permite alinear elementos de manera precisa sin generar saltos de línea innecesarios.

POSITION



Ideas Generales


  • El atributo position, tiene varias propiedade, haciendo que estas actuen sobres los elmentos, para asignarles una posion dentro del html.

  • La propiedad postion: static: actua en el elemento, se coloca en su flujo normal según el orden del documento HTML. No se puede mover o reposicionar utilizando estas propiedades.

  • La propiedad postion: relative: El elemento se coloca en su flujo normal, pero puedes desplazarlo horizontal y verticalmente utilizando las propiedades top, right, bottom y left.

  • La propiedad postion: absolute: el elemento se retira del flujo normal y se posiciona en relación a su primer padre con una posición distinta de static. Utiliza las propiedades top, right, bottom y left para posicionarlo en relación a ese padre.

  • La propiedad postion: fixed: El elemento se retira del flujo normal y se posiciona en relación a su primer padre con una posición distinta de static. Utiliza las propiedades top, right, bottom y left para posicionarlo en relación a ese padre.

  • La propiedad postion: sticky: Combina los aspectos de relative y fixed. Inicialmente, el elemento se comporta como relative. Sin embargo, cuando se desplaza la página y el elemento llega a un punto determinado en la ventana gráfica, se "pega" a ese punto y se desplaza con la página hasta que se alcanza el final del elemento o el contenedor.

Apuntes de Clase


La propiedad position, nos ayuda a ajustar los contenedores de una pagina web para su organizacion, en cual se nos ofrece 5 propiedades diferentes que tiene efectos diferentes: static, fixed, sticky, absolute y relative, cada una de ellas actua de diferente manere, una de ella se mantiene por encima de todo y en todo momento, la otra se posiciona en una esquina de una propiedad, la otra puede sufrir movienetos mediante los parametros top, right, bottom. left y la ultima combina dos propeidades como la relative y fixed, en el cual esta reacciona la hora de desplazarte por la pagina o extender esta, teniendo una reaccion a estos comportamiento.



RESUMEN

El atributo Position ofrece varias propiedades para controlar la ubicación de los elementos en una página. La propiedad position permite definir si un elemento se coloca en su flujo normal (static), si se desplaza en relación a su posición original (relative), si se fija a la ventana (fixed), si se posiciona en relación a su primer ancestro posicionado (absolute) o si combina las características de relative y fixed para crear un efecto de "pegado" (sticky). Estas propiedades, junto con top, right, bottom y left, ofrecen un gran control sobre la disposición de los elementos en una página web.



ANIMACION



Ideas Generales


  • CSS permite animar la mayoria de los elementos HTML sin usar JavaScript.

  • Cuando especifica estilos CSS dentro de la regla @keyframes, la animación cambiará del estilo actual al estilo nuevo

  • Para que una animación funcione, se debe vincular la animación a un elemento. div{ animation-name: animo1; nación en CSS @keyframes animo1 1 from fbackground-color: red;) to fbackground-color: yellow;)

  • animation-duration: especifica cuánto tiempo debe transcurrir una animación para completar un ciclo.

  • animation-iteration-count: especifica el número de veces que se debe reproducir una animación. Se expresa con números cerrados y se puede usar también infinito.

  • animation-timing-function: especifica la curva de velocidad de la animación

  • animation-fill-mode: especifica un estilo para el elemento cuando la animación no se está reproduciendo (antes de que comience, después de que termine o ambos)

Apuntes de Clase


Aprendimos que CSS nos permite crear animaciones increíbles sin necesidad de escribir una sola línea de JavaScript. Para crear una animación en CSS, seguimos estos pasos: Definimos los estilos clave: Usamos la regla @keyframes para especificar los diferentes estilos que queremos que tenga nuestro elemento a lo largo de la animación, vinculamos la animación a un elemento: Usamos la propiedad animation-name para asignar la animación que acabamos de definir a un elemento específico. Configuramos la duración: Con animation-duration indicamos cuánto tiempo tardará la animación en completar un ciclo. Repetimos la animación: La propiedad animation-iteration-count nos permite controlar cuántas veces se repite la animación, definimos el comportamiento al inicio y al final: Con animation-fill-mode podemos especificar qué estilo queremos que tenga el elemento antes de que comience, animation-timing-function nos permite ajustar la velocidad de la animación a lo largo del tiempo. Podemos elegir entre diferentes curvas de velocidad predefinidas o crear las nuestras.



RESUMEN

CSS nos permite crear animaciones sin usar JavaScript. Con CSS podemos hacer que los elementos de una página web se muevan, cambien de tamaño, color o forma de manera suave y gradual. Es como crear pequeñas películas dentro de nuestra página web.



REFERENCIAS AVANZADAS DE CSS3



Ideas Generales



  • Atributos ID, sirve para asignar un nombre en especifico a una tributo, como un indenticador


  • Pseudoclases, existe varias, :nth child(), nos permite buscar un hijo en especifico, :nth child(odd) y :nth child(even) e utilizan para seleccionar elementos con base en su posición en la secuencia de hijos, especficamente para alternar entre elementos impares y pares:, :nth first child, selecciona solo al primer hijo, :nth first child selecciona el ultimo hijo, :nth only child, afecta a un solo elemento simpre y cuando sea el ultimo y :nth not child esta hace una excepcion.


  • Los selectores de atributos funcionan en funcion del valor de uno de sus atributos, por ejemplo, selector de atributo (^=) selecciona elementos cuyo atributo comienza en un valor especifico, ($=) selecciona un atributo que termina en un valor especifico, (*=) selecciona aquellos que tiene una cadena especifica en cualquier posicion, selector de hijos directos (>) selecciona a los hijos inmediatos de un padre, (+) selector de relacion este elemento selecciona al que es su hermano inmediato, selector hermanos generales (~) selecciona a los hermanos anterior.

Apuntes de Clase


Los selectores CSS son herramientas poderosas que nos permiten seleccionar elementos HTML específicos para aplicarles estilos. Algunos de los tipos de selectores más comunes son: ID: Asigna un nombre único a un elemento, permitiendo seleccionarlo de forma precisa. Pseudoclases: Permiten seleccionar elementos basados en su estado o posición en el documento: :nth-child(): Selecciona elementos según su posición entre los hijos de su padre. :first-child, :last-child, :only-child: Seleccionan el primer, último o único hijo, respectivamente. :not(): Excluye elementos que cumplen ciertas condiciones. Selectores de atributos: Seleccionan elementos basados en el valor de sus atributos: ^=, $=, *=: Seleccionan elementos cuyo atributo comienza, termina o contiene un valor específico. Selectores de relaciones: Definen relaciones entre elementos: >: Selecciona hijos directos. +: Selecciona el siguiente hermano. ~: Selecciona todos los hermanos siguientes.



RESUMEN

ID: Señala un elemento único por su nombre. Por clase: Señala elementos que comparten una misma clase. Por etiqueta: Señala todos los elementos de un tipo específico (p.ej., todos los párrafos). Pseudoclases: Seleccionan elementos basados en su estado (hover, active) o posición (primero, último). Selectores de atributos: Seleccionan elementos basados en sus atributos. Selectores de relaciones: Definen relaciones entre elementos (padre, hijo, hermanos). Con estos selectores, podemos crear estilos muy precisos y combinarlos para lograr efectos visuales complejos.

  • Para indicar que usaras una table dentrol de HTML se usa la etiqueta < table>

  • La etiqueta < table> lleva consigo otras etiquetas como por ejemplo: thead que se usa para el encabezado de la tabla, th: celda del encabezado de la tabla, tbody: nos indica el cuerpo de la tabla tr: un reglon de la tabla, td: celda de datos, caption: usado para insertar captions, tfoot: pie de la pagina

  • La tablas antes se usaban para acomodar las posiciones dentro de un HTML ya que antes no habia forma de como acomodar las cosas dentro de un index

Apuntes de Clase



Las tablas HTML son una herramienta fundamental para organizar y presentar datos en una página web. Se utilizan para estructurar información en filas y columnas, creando una disposición clara y fácil de leer.
Elementos clave de una tabla: < table >: Define la tabla completa. < thead >: Encabezado de la tabla. < tbody >: Cuerpo de la tabla. < tr>: Fila de la tabla. < th>: Celda de encabezado. < td>: Celda de datos.
Aunque las tablas se utilizaban ampliamente para diseño en el pasado, hoy en día su uso principal es para presentar datos tabulares. Sin embargo, siguen siendo útiles para crear estructuras semánticas y mejorar la accesibilidad de las páginas web.




RESUMEN

La propiedad CSS display es fundamental para determinar cómo se posicionan y comportan los elementos en una página web. Esta propiedad, introducida en CSS 2.1, define el tipo de caja que cada elemento ocupa. Por defecto, los elementos HTML tienen un valor de visualización block (comienzan en una nueva línea y ocupan todo el ancho disponible) o inline (no comienzan en una nueva línea y solo ocupan el ancho necesario). El valor inline-block combina lo mejor de ambos mundos, permitiendo establecer un ancho y altura definidos, además de respetar los márgenes y rellenos, a diferencia de los elementos inline. Esta propiedad es invaluable para crear diseños flexibles y personalizados, ya que permite alinear elementos de manera precisa sin generar saltos de línea innecesarios.

ESPECIFICIDAD



Ideas Generales


  • La especificidad es el por el cual un navegador deciden que valores de CSS son mas relevantes

  • En la especificidad exiten jeraquias según la importacian: 1. Estilos inline, 2.ID´s, 3. Clases, Atributos y pseudos clases, 4. Elementos y pseudo-elementos

  • Selector Universal los valores heredados tiene una especifidad de 0.

  • !import, es una propiedad que anula cualquier otra declaracion y le da prioridad a esta el maximo, aunque no tiene nada que ver con la especificidad esta interactua con ellas

  • Cada selector tiene su lugar dentro de la jerarquiay existe 4 jerarquias

Apuntes de Clase


La especificidad determina qué regla "gana" y se aplica al elemento. Piensa en ella como una especie de puntaje que se asigna a cada regla. Cuanto más específico sea el selector de una regla (como un ID, una clase o un elemento), mayor será su puntaje.
La especificidad es una herramienta que ayuda al navegador a decidir qué estilo aplicar a un elemento cuando hay varias opciones. Cuanto más específico sea el selector, más probabilidades tendrá de ser aplicado



RESUMEN

La especificidad en CSS es como una competencia entre las reglas de estilo. Cuando varias reglas quieren "vestir" al mismo elemento HTML, la regla más específica es la que gana. Es decir, la regla que utilice selectores más precisos (como IDs o clases) tendrá prioridad sobre las que usen selectores más generales (como elementos)



HEADER



Ideas Generales


  • El Headear es una de las partes mas importante de una pagina web, pues es el elemento que los usuarios visualizan en primer lugar cuando llegan al proyecto.

  • La cabecera debe ser siempre ordenada y limpia para que pueda ser visualizada con un solo vistazo y no en lugar de causar confusion.

  • Los elementos principales de un headear son: logotipo, menú, buscador, datos de contacto, redes sociales, carrito de compras, botones destacados, selector de idioma o país, formulario de suscripción, etc.

  • El header tambien puede llevar un nav, para facilitar la navegacion del usuario dentro de la pagina web y este depende de como se organice el proyecto puede incluso llevar hasta dos.

  • El Header puede incluir botones destacados, se refiere a que puede tener algun boton que funcione de manera personalizada para el uso destacado.

Apuntes de Clase


¡Claro! Aquí tienes un resumen de la propiedad header en HTML: La etiqueta < header> en HTML: Un Encabezado Semántico La etiqueta < header> en HTML se utiliza para definir la cabecera de un documento o sección. Esta cabecera suele contener elementos introductorios o de navegación, como:
Título principal: El título más importante de la página o sección. Logo: La imagen representativa del sitio web. Menú de navegación: Los enlaces principales para navegar por el sitio. Formulario de búsqueda: Un campo para que los usuarios busquen contenido.
¿Por qué usar < header>? Semántica: Le proporciona a los motores de búsqueda y a los lectores de pantalla una clara indicación de la estructura del contenido. Organización: Agrupa elementos relacionados en una sola sección, lo que facilita la gestión del código y el diseño. Accesibilidad: Ayuda a las personas con discapacidades a comprender mejor la estructura de la página.



RESUMEN

La etiqueta < header> en HTML define la sección de encabezado de una página o sección. Generalmente se coloca al inicio y contiene elementos como el título principal, logo, menú de navegación y formulario de búsqueda. Su función principal es organizar y estructurar el contenido, mejorando la experiencia del usuario y el SEO. Al utilizar < header>, le indicas a los navegadores y motores de búsqueda que el contenido dentro de ella es el encabezado, lo que facilita la interpretación de la estructura de tu página.



FOOTER



Ideas Generales



  • El footer o pie de pagina es una sección olvidaba por muchos proyectos web, pero si es bien diseñado, puede ser muy útil y efectivo


  • En el footer puedes meter varias cosas, contacto, ubicacion, textos legales, sellos de confianza, CTA y formularios.


  • Los textos legales son muy importantes, porque incluyen las normas, que te protegen como dueño y al cliente en caso de cualquier incidente al momento de manejar su informacion.


  • Los formularios o newsletter para añadir una area de suscripciones.


  • Sellos de confianza, sirve para garantizar al usuario que la asociacion del pais encargado de proteger al consumidor, ha verificado que la pagina cumple con las normas de seguridad para garantizar una compra segura

Apuntes de Clase


La etiqueta < footer> se utiliza para definir el pie de página de un documento o sección. Esta sección, ubicada generalmente al final de una página, contiene información complementaria que no forma parte del contenido principal, pero que es relevante para el usuario. ¿Qué suele contener un < footer>? Información de copyright: Derechos de autor, licencias y demás detalles legales. Enlaces a políticas: Privacidad, términos de servicio, etc. Información de contacto: Dirección, teléfono, correo electrónico. Redes sociales: Enlaces a tus perfiles en diferentes plataformas. Mapa del sitio: Un enlace a una página que muestra la estructura completa del sitio. Créditos: Agradecimientos a colaboradores, diseñadores, etc.



RESUMEN

El footer define el pie de página, al final de la página o sección. Incluye información complementaria como copyright, políticas de privacidad, enlaces a redes sociales y mapa del sitio. Ambas etiquetas proporcionan una estructura semántica clara a tus documentos HTML, mejorando la organización y accesibilidad de tu sitio web.



COLORES WEB



Ideas Generales



  • El seleccionar una buena paleta de colores, segÚn para que va destinido nuestro sitio puede ser de gran ayuda, ya que el usario tiende a relacionar los colores con algun tema especifico y estos generan confianza


  • Hay varas formas de representar los colores en HTML, desde hexadecimal en donde un conjunto de numero y letras nos generan un color


  • Tenemos dos formatos en donde RGB es la mezcla de los 3 colores primarios, red, green y blue, el otro que tenemos es RGBA donde nuevamente se presenta la regla de red, blue y green, solo que ahora la a representa la alpha que nos indica el nivel de transpariencia.

  • Otros formatos que funcionan de la misma manera son los HSL, HSLA en estos juegan con diferentes parametros, pero hacen la misma funcion.

  • Los colores son lo mas escencial y por eso hay que tener una armonia entro estos, que ciertos colores se asocian algún ámbito, no signifca que debes mezclar todos lo colores.

Apuntes de Clase


El color en el diseño web es fundamental. Influye en las emociones, la percepción y las decisiones de los usuarios. Los colores pueden transmitir sensaciones, crear identidad de marca y guiar la atención del visitante. Los formatos de color más comunes utilizados en CSS son:
Hexadecimal: Utiliza una combinación de seis dígitos hexadecimales (#000000 a #FFFFFF) para representar un color. RGB: Especifica un color combinando los valores de rojo, verde y azul (por ejemplo, rgb(255, 0, 0) para el rojo). RGBA: Similar a RGB, pero incluye un cuarto canal para la transparencia (por ejemplo, rgba(255, 0, 0, 0.5)). HSL: Define un color utilizando el tono, la saturación y la luminosidad (por ejemplo, hsl(0, 100%, 50%) para el rojo). HSL: Define un color utilizando el tono, la saturación y la luminosidad (por ejemplo, hsl(0, 100%, 50%) para el rojo). HSLA: Similar a HSL, pero incluye un cuarto canal para la transparencia.



RESUMEN

El color es fundamental en el diseño web. Influye en las emociones, la percepción y las decisiones de los usuarios. Los colores pueden transmitir sensaciones, crear identidad de marca y guiar la atención. Los formatos de color más comunes en CSS son: hexadecimal, RGB, RGBA, HSL y HSLA. La elección adecuada de los colores es clave para un diseño atractivo y efectivo.
